Detailed Description

This file exposes the API of the OsciQueue concurrent queue implementation. An example of use of this API is provided in benchmarks/osciqueuebench.c file.

Author
Nikolaos D. Kallimanis (nkall.nosp@m.ima@.nosp@m.gmail.nosp@m..com)

For a more detailed description see the original publication: Panagiota Fatourou, and Nikolaos D. Kallimanis. "Lock Oscillation: Boosting the Performance of Concurrent Data Structures", In Proceedings of the 21st International Conference on Principles of Distributed Systems (Opodis), 2017.

struct SynchOsciQueueStruct

SynchOsciQueueStruct stores the state of an instance of the OsciQueue concurrent queue implementation. SynchOsciQueueStruct should be initialized using the OsciQueueStructInit function.

◆ SynchOsciQueueThreadState

struct SynchOsciQueueThreadState

SynchOsciQueueThreadState stores each thread's local state for a single instance of OsciQueue. For each instance of OsciQueue, a discrete instance of SynchOsciQueueThreadState should be used.

Data Fields

SynchOsciThreadState dequeue_thread_state An SynchOsciThreadState struct for the instance of Osci that serves the dequeue operations.

void synchOsciQueueInit ( SynchOsciQueueStruct *  queue_object_struct,
uint32_t  nthreads,
uint32_t  fibers_per_thread 
)

This function initializes an instance of the OsciQueue concurrent queue implementation.

This function should be called once (by a single thread) before any other thread tries to apply any enqueue or dequeue operation.

Parameters
queue_object_structA pointer to an instance of the OsciQueue concurrent queue implementation.
nthreadsThe number of threads that will use the OsciQueue concurrent queue implementation.
fibers_per_threadThe number of fibers per Posix thread.
