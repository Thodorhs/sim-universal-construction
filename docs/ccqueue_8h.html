Detailed Description

This file exposes the API of the CC-Queue concurrent queue implementation. An example of use of this API is provided in benchmarks/ccqueuebench.c file.

Author
Nikolaos D. Kallimanis (nkall.nosp@m.ima@.nosp@m.gmail.nosp@m..com)

For a more detailed description see the original publication: Panagiota Fatourou, and Nikolaos D. Kallimanis."Revisiting the combining synchronization technique". ACM SIGPLAN Notices. Vol. 47. No. 8. ACM, PPoPP 2012.

Data Structure Documentation

◆ SynchCCQueueStruct

struct SynchCCQueueStruct

SynchCCQueueStruct stores the state of an instance of the CC-Queue concurrent queue implementation. SynchCCQueueStruct should be initialized using the synchCCQueueStructInit function.

◆ SynchCCQueueThreadState

struct SynchCCQueueThreadState

SynchCCQueueThreadState stores each thread's local state for a single instance of CC-Queue. For each instance of CC-Queue, a discrete instance of SynchCCQueueThreadState should be used.

Data Fields
SynchCCSynchThreadState enqueue_thread_state A SynchCCSynchThreadState struct for the instance of CC-Synch that serves the enqueue operations.
SynchCCSynchThreadState dequeue_thread_state A SynchCCSynchThreadState struct for the instance of CC-Synch that serves the dequeue operations.

uint32_t  nthreads

This function initializes an instance of the CC-Queue concurrent queue implementation. This function should be called once (by a single thread) before any other thread tries to apply any enqueue or dequeue operation.

Parameters
queue_object_structA pointer to an instance of the CC-Queue concurrent queue implementation.
nthreadsThe number of threads that will use the CC-Queue concurrent queue implementation.
