Detailed Description

This file exposes the API of the DSMSynch combining object. An example of use of this API is provided in benchmarks/dsmsynchbench.c file.

Author
Nikolaos D. Kallimanis (nkall.nosp@m.ima@.nosp@m.gmail.nosp@m..com)

For a more detailed description see the original publication: Panagiota Fatourou, and Nikolaos D. Kallimanis."Revisiting the combining synchronization technique". ACM SIGPLAN Notices. Vol. 47. No. 8. ACM, PPoPP 2012.

◆ SynchDSMSynchNode

struct SynchDSMSynchNode

SynchDSMSynchNode stores the data of an announced request.

Data Fields
struct SynchDSMSynchNode * next Pointer to the next request that has been announced.
ArgVal arg_ret This variable stores the argument of the request and the return value after the request is applied.
uint32_t pid The pid of the thread that announced this request.
volatile uint32_t locked Whenever it is equal to false, the thread is the combiner; otherwise the thread waits until a combiner apply its request.
volatile uint32_t completed If true, the request is applied and the thread returns its return value.
char align[PAD_CACHE(sizeof(SynchHalfDSMSynchNode))] Padding space.

This function initializes an instance of the DSMSynch combining object.

This function should be called once (by a single thread) before any other thread tries to apply any request by using the synchDSMSynchApplyOp function.

Parameters
lA pointer to an instance of the DSMSynch combining object.
